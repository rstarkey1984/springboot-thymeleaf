# 9교시 - 기존 화면을 Thymeleaf 표현식으로 전환

## 학습 목표
- 1~7교시에서 작성한 화면 코드를 표현식 기반으로 전환할 수 있다.
- `Model` 데이터를 활용해 하드코딩 영역을 제거할 수 있다.

## 학습 범위
- 정적 문구를 `th:text`로 전환
- 링크를 `th:href`로 전환
- 입력값 유지(`th:value`) 적용
- 조건 메시지(`th:if`) 적용
- 반복 출력(`th:each`) 적용

## 전환 원칙
- 1~7교시: `th:if`, `th:text`는 기능 구현과 함께 적극 사용
- 8교시: 표현식 연습 전용 페이지에서 반복 훈련
- 9교시: 기존 코드에 나머지 표현식(`th:each`, `th:classappend`, `th:value`, `th:href`)을 확장 적용

## 실습 절차
1. `login.html`의 하드코딩 텍스트를 `th:text`로 바꾼다.
2. 링크를 `href`에서 `th:href`로 바꾼다.
3. 로그인 입력값 유지에 `th:value`를 적용한다.
4. 성공/실패/로그아웃 메시지를 `th:if`로 제어한다.
5. 에러 목록이 있다면 `th:each`로 출력한다.

## 확인 문제
- [ ] 하드코딩 텍스트를 `th:text`로 변환했는가?
- [ ] 링크를 `th:href`로 변환했는가?
- [ ] 메시지 출력을 `th:if`로 전환했는가?
- [ ] 목록 출력에 `th:each`를 적용했는가?

## 실습 코드

### 전환 전 예시 (일부)
```html
<h1>로그인</h1>
<p>로그인 성공</p>
<a href="/signup">회원가입으로 이동</a>
```

### 전환 후 예시 (일부)
```html
<h1 th:text="${pageTitle}">로그인</h1>
<p th:if="${successMessage}" th:text="${successMessage}"></p>
<a th:href="@{/signup}">회원가입으로 이동</a>
```

## 정리
표현식 전환을 완료하면 화면이 데이터 중심 구조가 되어,
기능 확장과 유지보수가 쉬워진다.
