# 7교시 - 로그인/회원가입 화면 개선과 사용자 피드백

## 학습 목표
- 로그인/회원가입 화면의 사용성을 개선할 수 있다.
- 성공/실패 메시지를 화면에 표시할 수 있다.
- 사용자가 다음 행동을 이해하기 쉬운 UI를 구성할 수 있다.

## 학습 범위
- 로그인 성공/실패 메시지 처리
- 회원가입 완료 메시지 처리
- 로그아웃/세션 메시지 처리
- 화면 간 이동 링크 정리

## 핵심 개념
- 사용자 피드백은 기능 완성도에 직접적인 영향을 준다.
- 처리 결과를 URL 파라미터 또는 모델 데이터로 전달할 수 있다.
- 화면 문구는 짧고 명확하게 작성한다.

## 실습 절차
1. 로그인 성공/실패 메시지 영역을 추가한다.
2. 회원가입 완료 메시지 흐름을 추가한다.
3. 화면 하단 이동 링크를 정리한다.
4. 실제 동작을 확인한다.

## 확인 문제
- [ ] 로그인 성공/실패 메시지를 구분해 출력할 수 있는가?
- [ ] 회원가입 완료 후 안내 문구를 표시할 수 있는가?
- [ ] 로그인/회원가입 화면 이동 흐름을 설명할 수 있는가?

## 실습 코드

### controller/AuthController.java (일부)
```java
@PostMapping("/signup")
public String signupSubmit(
        @RequestParam String username,
        @RequestParam String password,
        @RequestParam String name
) {
    authService.signup(username, password, name);
    return "redirect:/login?signupSuccess";
}

@PostMapping("/login")
public String loginSubmit(
        @RequestParam String username,
        @RequestParam String password
) {
    if (authService.login(username, password)) {
        return "redirect:/login?success";
    }
    return "redirect:/login?error";
}
```

### templates/login.html
```html
<!doctype html>
<html lang="ko" xmlns:th="http://www.thymeleaf.org">
<head>
  <meta charset="UTF-8">
  <title>로그인</title>
</head>
<body>
<h1>로그인</h1>

<p th:if="${param.signupSuccess}">회원가입이 완료되었습니다. 로그인해 주세요.</p>
<p th:if="${param.success}">로그인 성공</p>
<p th:if="${param.error}">아이디 또는 비밀번호가 올바르지 않습니다.</p>
<p th:if="${param.logout}">로그아웃되었습니다.</p>

<form method="post" action="/login">
  <label>아이디 <input type="text" name="username"></label><br>
  <label>비밀번호 <input type="password" name="password"></label><br>
  <button type="submit">로그인</button>
</form>

<a href="/signup">회원가입으로 이동</a>
</body>
</html>
```
