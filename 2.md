# 2교시 - 회원가입 화면과 Controller 구현

## 학습 목표
- 회원가입 화면(`signup.html`)을 작성할 수 있다.
- 회원가입 GET/POST 매핑을 Controller에 구현할 수 있다.

## 학습 범위
- `GET /signup` 화면 반환
- `POST /signup` 요청 처리
- 폼 데이터 바인딩 기초

## 핵심 개념
- GET은 화면을 보여준다.
- POST는 사용자가 입력한 데이터를 처리한다.
- Controller는 흐름만 담당하고, 세부 검증은 Service에서 담당한다.

## 실습 절차
1. `signup.html` 템플릿을 작성한다.
2. `GET /signup` 매핑을 작성한다.
3. `POST /signup` 매핑을 작성한다.
4. 가입 완료 후 로그인 페이지로 리다이렉트한다.

## 확인 문제
- [ ] `/signup` GET/POST의 역할을 구분할 수 있는가?
- [ ] 회원가입 폼 필드를 구성할 수 있는가?
- [ ] 가입 후 리다이렉트 흐름을 구현할 수 있는가?

## 실습 코드

### controller/AuthController.java
```java
package com.example.lms.controller;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestParam;

@Controller
public class AuthController {

    @GetMapping("/signup")
    public String signupForm() {
        return "signup";
    }

    @PostMapping("/signup")
    public String signupSubmit(
            @RequestParam String username,
            @RequestParam String password,
            @RequestParam String name
    ) {
        // 다음 교시에서 Service 호출로 확장
        return "redirect:/login";
    }
}
```

### templates/signup.html
```html
<!doctype html>
<html lang="ko" xmlns:th="http://www.thymeleaf.org">

  <label>아이디 <input type="text" name="username"></label><br>

</body>
</html>
```

## 정리
회원가입 기능은 화면(GET)과 처리(POST)를 분리해 구현한다.
다음 교시에서는 로그인 화면/처리 흐름을 같은 방식으로 구현한다.
