# 10교시 - 통합 점검과 최종 정리 (회원가입/로그인)

## 학습 목표
- 회원가입/로그인/로그아웃 전체 흐름을 통합 실행으로 점검할 수 있다.
- 세션 생성/유지/종료 동작을 검증할 수 있다.
- 계층 분리(Controller-Service-Repository) 상태를 확인할 수 있다.
- 예외 처리와 사용자 피드백 동작을 검증할 수 있다.

## 학습 범위
- 실행 점검
- 기능 점검
- 세션 점검
- 구조 점검
- 최종 체크리스트

## 핵심 내용
- 기능이 동작하는 것과 구조가 올바른 것은 별도 점검이 필요하다.
- 예외 흐름은 정상 흐름과 동일하게 중요하다.
- 사용자 피드백 문구는 실제 사용성에 영향을 준다.

## 실습 절차
1. 프로젝트를 실행한다.
2. 회원가입을 수행한다.
3. 로그인 성공/실패를 각각 확인한다.
4. 예외 상황을 유도해 오류 페이지를 확인한다.
5. 최종 체크리스트를 완료한다.

## 최종 체크리스트
- [ ] `./gradlew bootRun` 실행 성공
- [ ] `GET /signup` 화면 정상 출력
- [ ] `POST /signup` 처리 후 로그인 페이지 이동
- [ ] `GET /login` 화면 정상 출력
- [ ] `POST /login` 성공/실패 분기 정상 동작
- [ ] 로그인 성공 시 세션(`LOGIN_USER`) 생성 확인
- [ ] `GET /logout` 호출 시 세션 종료 확인
- [ ] `AppException` 발생 시 오류 화면 출력
- [ ] Controller-Service-Repository 계층 분리 유지

## 실습 코드

### 실행 명령
```bash
./gradlew clean build
./gradlew bootRun
```

### 점검 URL
```text
/signup
/login
/logout
```

### 점검용 시나리오 예시
```text
1) 회원가입 입력값 정상 -> 가입 성공 -> Flash Attribute(signupSuccessMessage)
2) 로그인 아이디/비밀번호 정상 -> Flash Attribute(successMessage) + 세션 생성(LOGIN_USER)
3) /logout 호출 -> Flash Attribute(logoutMessage) + 세션 종료
4) 로그인 아이디/비밀번호 오류 -> Flash Attribute(errorMessage)
5) 회원가입 필수값 누락 -> AppException -> 오류 페이지
```

## 정리
회원가입/로그인 기능은 단순 화면 구현이 아니라,
요청 처리, 검증, 데이터 접근, 예외 처리, 사용자 피드백까지 포함한 전체 흐름으로 완성된다.
