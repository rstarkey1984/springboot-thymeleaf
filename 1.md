# 1교시 - 프로젝트 시작과 로그인/회원가입 흐름 이해

## 학습 목표
- Spring Boot + Thymeleaf 실습 프로젝트를 시작할 수 있다.
- 로그인/회원가입 기능의 전체 흐름을 설명할 수 있다.

## 학습 범위
- 프로젝트 기본 구조 확인
- 로그인/회원가입 URL 설계
- Controller와 템플릿 연결 개요

## 핵심 개념
- `GET /signup` : 회원가입 화면 요청
- `POST /signup` : 회원가입 데이터 처리
- `GET /login` : 로그인 화면 요청
- `POST /login` : 로그인 데이터 처리

## 실습 절차
1. 프로젝트 루트 구조를 확인한다.
2. `controller`, `service`, `repository`, `templates` 역할을 정리한다.
3. 로그인/회원가입 URL 4개를 설계한다.
4. 템플릿 파일 경로를 결정한다.

## 확인 문제
- [ ] 로그인/회원가입 URL 4개를 구분할 수 있는가?
- [ ] GET과 POST의 역할 차이를 설명할 수 있는가?
- [ ] 템플릿 경로를 URL 기준으로 설계할 수 있는가?

## 실습 코드

### URL 설계 예시
```text
GET  /signup
POST /signup
GET  /login
POST /login
```

### 기본 패키지 구조 예시
```text
com.example.lms
 ├─ controller
 ├─ service
 ├─ repository
 ├─ common
 └─ security
```

### 템플릿 경로 예시
```text
src/main/resources/templates/signup.html
src/main/resources/templates/login.html
src/main/resources/templates/error/app-error.html
```

다음 교시부터 회원가입 화면과 Controller를 실제로 구현한다.
