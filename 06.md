# 6교시 - AppException과 전역 예외 처리

## 학습 목표
- `AppException`으로 예외를 통일할 수 있다.
- 전역 예외 처리(`@ControllerAdvice`)를 적용할 수 있다.
- 예외 발생 시 사용자에게 오류 화면을 제공할 수 있다.

## 학습 범위
- `AppException` 클래스 작성
- Service 예외를 `AppException`으로 변경
- `GlobalExceptionHandler` 작성
- 오류 템플릿 연결

## 핵심 개념
- Service 계층에서 비즈니스 예외를 명확히 표현한다.
- Controller는 try-catch를 최소화하고 전역 처리에 위임한다.
- 예외 메시지는 사용자에게 이해 가능한 형태로 전달한다.

## 실습 절차
1. `AppException` 클래스를 생성한다.
2. `AuthService`의 `RuntimeException`을 `AppException`으로 교체한다.
3. `GlobalExceptionHandler`를 작성한다.
4. 오류 페이지 템플릿을 작성하고 동작을 확인한다.

## 실습 코드

### common/AppException.java
```java
package com.example.lms.common;

public class AppException extends RuntimeException {
    public AppException(String message) {
        super(message);
    }
}
```

### service/AuthService.java (일부)
```java
import com.example.lms.common.AppException;

public void signup(String username, String password, String name) {
    validateSignup(username, password, name);

    if (authRepository.existsByUsername(username)) {
        throw new AppException("이미 사용 중인 아이디입니다.");
    }

    authRepository.save(new Member(username, password, name));
}

private void validateSignup(String username, String password, String name) {
    if (username == null || username.isBlank()) {
        throw new AppException("아이디는 필수입니다.");
    }
    if (password == null || password.length() < 4) {
        throw new AppException("비밀번호는 4자 이상이어야 합니다.");
    }
    if (name == null || name.isBlank()) {
        throw new AppException("이름은 필수입니다.");
    }
}
```

### controller/advice/GlobalExceptionHandler.java
```java
package com.example.lms.controller.advice;

import com.example.lms.common.AppException;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.ControllerAdvice;
import org.springframework.web.bind.annotation.ExceptionHandler;

@ControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(AppException.class)
    public String handleAppException(AppException e, Model model) {
        model.addAttribute("message", e.getMessage());
        return "error/app-error";
    }
}
```

### templates/error/app-error.html
```html
<!doctype html>
<html lang="ko" xmlns:th="http://www.thymeleaf.org">
<head>
  <meta charset="UTF-8">
  <title>오류 안내</title>
</head>
<body>
<h2>요청 처리 중 오류가 발생했습니다.</h2>
<p th:text="${message}"></p>
<a href="/login">로그인으로 이동</a>
</body>
</html>
```

## 정리
예외를 `AppException`으로 통일하고 전역 처리 구조를 적용하면,
코드 중복을 줄이고 사용자 오류 안내를 일관되게 제공할 수 있다.
